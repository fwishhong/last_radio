# 《末日电台：旧体育馆守夜》正式版策划 V1

英文名：Last Radio  
中文名暂定：《末日电台：旧体育馆守夜》

本文档是正式制作蓝图，不是原型备忘。它定义项目方向、核心循环、系统边界、章节结构和近期制作范围。若与旧 `LAST_RADIO_V2_DESIGN.md` 冲突，以本文档为当前正式方向。

## 0. 当前硬性玩法口径

- 一夜叙事时间固定为 22:00 到 06:00，共 8 小时；玩法时间按关卡压缩，第 1 夜 60 秒，第 2 夜 120 秒，第 3 夜起进入 180 秒级。
- 第一夜不能是长时间等待，必须有多次清晰危机：正门、左窗、发电机、黎明前补压。
- 玩家开局只拥有一个可控主角。Nora、Elias 等角色不是开局队伍成员，而是听到广播、熬过夜晚、完成电台回应后逐渐加入的辅助 NPC。
- 所有热点必须通过提示圆圈、状态横条和颜色/日志变化告诉玩家哪里正在衰减、哪里需要干涉。
- 后期加难度优先增加事件密度、路线冲突、白天代价和暴露度后果，不优先增加无限新热点或拉长等待时间。

## 1. 游戏定位

《末日电台：旧体育馆守夜》是一款 **白天决策 + 夜晚场地守卫 + 叙事生存战役** 游戏。

玩家是旧体育馆避难点的值班员。白天安排加固、搜救、搜刮和广播；夜晚亲自在体育馆里奔走，守住门窗、电力、天线、电台和避难者，撑到下一次天亮。

核心体验不是“管理很多数字”，而是：

- 我只有一个身体，所以必须决定先去哪。
- 我救的人越多，体育馆越有希望，也越容易被看见。
- 白天的选择会在夜晚变成具体压力。
- 电台不是菜单，它是希望、诱饵和风险源。

## 2. 标题方向

正式推荐名：

**《末日电台：旧体育馆守夜》**

理由：

- “末日电台”直接、好记、和英文 Last Radio 对应度高。
- “旧体育馆守夜”提供独特地点和玩法气质，避免标题过泛。
- 后续章节可以扩展为《末日电台：某某章节》，而 Last Radio 保持英文主品牌。

备选：

- 《最后一频》：记忆点强，但含义不够直观。
- 《天亮前的电台》：贴合循环，但品牌感弱。
- 《黎明电台》：更温暖，但末日压迫感不足。
- 《旧体育馆电台》：地点明确，但标题偏朴素。

## 3. 世界规则

白天不是安全，只是危险形态不同。

夜晚：

- 尸群受声音、灯光、热源和广播吸引。
- 门、窗、后门等实体边界会被直接冲击。
- 玩家必须在场地里移动，处理具体热点。

白天：

- 强光和环境噪声让尸群分散，直接围攻减少。
- 外出搜救、搜刮和修复仍有风险。
- 风险通过白天卡牌结算，不做实时操作。
- 白天做出的选择会影响夜晚的热点压力、资源、暴露度和 NPC 状态。

这条规则解释为什么白天不被围攻，也让白天系统从“升级菜单”变成“冒险决策”。

## 4. 核心循环

每一夜前后分为四段。

### 4.1 白天报告

展示上一夜结果：

- 哪些门窗撑住了。
- 电力、电台、天线是否稳定。
- 避难者状态是否变化。
- Nora、Elias、Victor 等角色的剧情反馈。
- 失败时重打当前夜，不推进日期。

失败原则：

- 普通难度失败后重打当前夜。
- 不永久扣人、不永久锁死剧情。
- 可以保留失败报告和策略提示，帮助玩家换打法。

### 4.2 白天决策

玩家从 3 张左右的卡牌中选择当天重点。正式版白天卡牌不只升级，分为四类：

- 加固：修门、补窗、检修发电机、固定天线。
- 搜救：寻找幸存者、接应队伍、回应求救信号。
- 搜刮：寻找木板、零件、电池、药品、燃料。
- 广播：公开坐标、保持沉默、转发求救、伪装信号。

每张卡必须有收益和代价。不能只有“数值变好”。

示例：

- 搜救孩子：增加人口和信任，但消耗药品，并提高今晚噪音事件概率。
- 公开广播：提高影响力和外部援助概率，但增加暴露度。
- 保持沉默：降低暴露度，但可能错过幸存者或物资线索。
- 检修发电机：降低今晚掉电速度，但消耗零件。

### 4.3 夜晚守卫

夜晚是当前原型已经成立的主玩法。

玩家操作一个主角：

- 点击热点。
- 主角走向目标。
- 到达后自动修复、封堵、接通或处理。
- 移动时间本身是代价。

夜晚目标：

- 撑到天亮。
- 不能让关键门窗破防太久。
- 不能让电力、电台或天线在关键时刻失效。
- 某些夜晚有额外目标，例如接通 Elias、回应 Victor、保护医务角。

### 4.4 天亮结算

天亮后进入报告：

- 成功推进剧情。
- 解锁新地点、新角色、新热点或新白天卡牌。
- 根据白天选择和夜晚表现改变资源、暴露度、NPC 状态。

## 5. 夜晚系统

### 5.1 主角

正式版仍然只允许玩家直接控制一个主角。

原因：

- 强化“只有一个身体”的紧张感。
- 避免变成 RTS 或多人调度游戏。
- 让移动路线和优先级判断成为核心。

主角能力：

- 四方向行走动画。
- 点击热点后自动移动。
- 到达后执行处理动作。
- 处理动作通过热点状态条反馈。

### 5.2 辅助 NPC

NPC 是辅助，不是可直接控制单位。

Nora：

- 优先处理窗户和医务相关事件。
- 擅长保护避难者、安抚恐慌。
- 她的升级应围绕窗户、医务、儿童和避难者信任。

Elias：

- 优先处理电台、天线、发电机。
- 擅长信号、电力和工具。
- 他的升级应围绕电台接通、天线稳定、自动维修。

Victor：

- 不在体育馆内。
- 通过电台提供坐标、路线、警告和剧情压力。
- 后期他的广播会让体育馆得到援助，也会带来追踪风险。

### 5.3 热点

第一章核心热点：

- 正门：基础防线，失守会导致失败。
- 左窗：早期教学压力点。
- 右窗：引入 Nora 自动协助。
- 发电机：影响灯光、电台、医务角和夜晚视野。
- 电台：回应外部角色，推进剧情和获得支援。
- 天线：保证电台信号，后期与暴露度关联。
- 后门：中期新增防线，靠近发电机。
- 医务角：处理伤员和恐慌，不一定立即失败，但会占用时间。
- 仓库：木板和应急封堵资源来源。

后期不靠无限增加热点加难度。难度来自：

- 同一热点承担多重意义。
- 多个系统同时压迫。
- 白天选择制造夜晚后果。
- 暴露度让事件更密、更险。

### 5.4 状态显示

每个可处理热点必须有：

- 圆形提示圈：表示当前可注意对象。
- 横向状态条：表示耐久、稳定度、信号或电量。
- 颜色变化：绿色稳定，黄色警告，红色危险，蓝色特殊/电台。
- 当前目标高亮。
- 短日志说明发生了什么。

玩家应能在 5 秒内判断：

- 哪里危险。
- 哪里正在衰减。
- 主角当前去哪。
- 哪个点需要优先处理。

## 6. 白天系统

正式版白天不是纯升级树，而是“今天把有限行动投入哪里”。

### 6.1 白天卡牌类型

加固卡：

- 直接改善某个热点。
- 通常消耗木板、零件、电池或燃料。
- 对当晚效果明显。

搜救卡：

- 增加人口、NPC、剧情线索或信任。
- 可能带来伤员、噪音、感染疑虑、追踪风险。

搜刮卡：

- 获得木板、零件、药品、电池、燃料。
- 可能消耗 NPC 疲劳或提高暴露度。

广播卡：

- 改变外部关系、影响力和暴露度。
- 可能解锁路线、援助、敌对事件或 Victor 剧情。

### 6.2 白天资源

资源必须少而有用，避免表格化。

推荐正式资源：

- 木板：封堵门窗、白天加固、夜晚应急。
- 零件：修发电机、天线、电台。
- 电池/燃料：影响断电风险和广播能力。
- 药品：处理医务角、搜救伤员。
- 人手/疲劳：NPC 白天行动会影响夜晚效率。
- 暴露度：广播、灯光、搜救和人口增长会提高被发现概率。
- 信任：影响避难者稳定、NPC 冲突、某些选择是否可用。

### 6.3 暴露度

暴露度是正式版后期难度的核心解释。

暴露度上升来源：

- 公开广播坐标。
- 夜晚长时间开强光。
- 搜救带回大量幸存者。
- 电台功率升级。
- 外部势力追踪。

暴露度影响：

- 夜晚冲击频率上升。
- 同时危机概率上升。
- 后门、天线、电台更容易成为目标。
- 某些白天卡牌风险提高。

暴露度下降方式：

- 保持沉默。
- 伪装信号。
- 关闭部分灯光。
- 使用诱饵广播。
- 完成特定剧情路线。

## 7. 难度曲线

正式版不应只靠数值变大。

阶段 1：理解场地

- 只有正门、左窗、发电机。
- 教会移动、修复、状态条、天亮目标。

阶段 2：同伴协作

- 解锁右窗、Nora。
- 玩家学会信任辅助 NPC，但不能完全依赖她。

阶段 3：电台与外部世界

- 解锁电台、Elias、Victor。
- 玩家在接通电台和守门窗之间取舍。

阶段 4：信号风险

- 解锁天线。
- 电台不再只是好事，信号会暴露体育馆。

阶段 5：白天后果

- 搜救、搜刮、广播开始改变夜晚事件。
- 玩家开始为未来冒险，而不是只优化今晚。

阶段 6：多系统缠绕

- 发电机影响灯光、天线、电台、医务角。
- 后门、医务角、仓库加入，但热点数量不再无限增加。

阶段 7：最终广播

- 玩家决定体育馆是继续隐藏、成为灯塔，还是作为诱饵换取更多人活下来。

## 8. 章节结构

正式版建议采用章节制，而不是一口气做无限长战役。

### 第一章：旧体育馆

长度：10 夜左右。  
目标：建立避难点，接通电台，形成基础同伴组。

核心内容：

- 主角、Nora、Elias、Victor 登场。
- 体育馆所有基础热点解锁。
- 白天卡牌从单纯升级扩展到搜救、搜刮、广播。
- 结尾让体育馆成为被看见的坐标。

### 第二章：周边街区

长度：7-10 夜。  
目标：把白天行动扩展到周边节点。

核心内容：

- 白天搜救和搜刮更重要。
- 体育馆夜晚仍是主舞台，但事件来自外部选择。
- 引入外部势力和路线判断。

### 第三章：信号网

长度：7-10 夜。  
目标：多个幸存点接入电台，玩家决定帮谁、放弃谁。

核心内容：

- 广播选择成为战略核心。
- 暴露度和影响力互相拉扯。
- Victor 的位置与代价逐渐浮现。

### 第四章：最后广播

长度：最终章，3-5 夜或连续危机。  
目标：回收玩家路线，决定电台命运。

可能结局方向：

- 灯塔结局：体育馆成为互助网络核心，但代价沉重。
- 隐蔽结局：体育馆活下来，但外部影响有限。
- 诱饵结局：电台吸引追踪，换取其他幸存点撤离。
- 失声结局：体育馆存活，电台沉默，城市不再回应。

## 9. 第一章制作范围

第一章是近期正式制作目标。

必须完成：

- 10 夜可完整通关。
- 1 个夜晚主场景：旧体育馆避难区。
- 1 个可控主角，四方向行走关键帧动画。
- Nora、Elias 两名半自动辅助 NPC。
- Victor 电台剧情线。
- 8-9 个热点：正门、左窗、右窗、发电机、电台、天线、后门、医务角、仓库。
- 白天 3 选 1 卡牌，逐步加入加固、搜救、搜刮、广播。
- 中文主语言，i18n key 结构预留。
- Windows 可试玩构建。

第一章暂不做：

- 自由大地图移动。
- 多角色直接操控。
- 武器射击或实时战斗。
- 复杂装备栏。
- 大量基地房间经营。

## 10. 美术与音频方向

美术关键词：

- 旧体育馆。
- 临时避难。
- 冷光、手电、发电机暖光。
- 白天灰尘和疲惫，夜晚压迫和红色危险。
- 怪物少露全貌，多通过门窗、影子、手、撞击、声音表达。

音频关键词：

- 电台底噪。
- 远处体育馆结构声。
- 发电机低频。
- 门窗撞击。
- 天线风声。
- 天亮时短暂释放。

音频原则：

- 警告音要可识别，但不能吵到像 UI 报警器。
- 电台音效要有叙事性。
- 音乐分白天、前期夜晚、后期夜晚、最终夜。

## 11. 文本与多语言规范

当前正式语言：中文。  
英文名和角色名可保留英文：Last Radio、Nora、Elias、Victor。

文本原则：

- UI 短句优先。
- 日志一行内表达清楚。
- 剧情文本服务选择和压力，不写长篇小说。
- 角色语气稳定：
  - Nora：直接、保护欲强、少废话。
  - Elias：技术细节、紧张但可靠。
  - Victor：远方、克制、带牺牲感。

多语言结构：

- 所有 UI 文案进入 i18n 文件。
- 剧情文本后续也要从脚本迁出。
- key 使用功能前缀，例如 `ui_`、`log_`、`day_`、`night_`、`report_`。
- 当前 GDScript 中的中文兜底只作为临时保护，不作为长期方案。

## 12. 工具与 MCP 策略

当前不需要安装新的 MCP 或 skill 才能继续。

已有 `last-radio-godot` skill 足够覆盖：

- 项目结构。
- NightShift 主场景。
- Godot 测试命令。
- 截图检查流程。
- 美术资源路径。

后续可考虑：

- Godot MCP：当需要大量编辑场景节点、动画、UI 布局时使用。
- 项目设计 skill：当正式策划稳定后，可以建立 `last-radio-design`，固定世界规则、文本风格、资源命名和章节结构。

原则：

- 设计阶段优先写清楚约束。
- 实装阶段优先本地测试和截图。
- 工具只在能减少重复劳动时引入，不为了“看起来专业”而安装。

## 13. 下一步制作顺序

建议下一阶段按以下顺序推进：

3. 加入资源：木板、零件、电池/燃料、药品、暴露度、信任。

6. 做第一章前 3 夜正式垂直切片。
7. 导出 Windows 内测包。

## 14. 当前设计结论

正式版不再是单纯“晚上点热点”的小原型，也不回到旧 V2 那种偏基地经营的菜单游戏。

最终方向是：

**白天做有代价的生存决策，夜晚用一个真实在场的主角承担这些决策的后果。**

这就是 Last Radio 和普通末日经营游戏、普通塔防游戏的区别。
