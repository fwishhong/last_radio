# AGENTS.md

Last Radio: Old Stadium Watch / 末日电台：旧体育馆守夜 — a 10-night resource-and-positioning
defense game set in an abandoned stadium on the edge of a dead city. Solo-developed
Godot 4.3-stable GDScript project targeting Steam release (zh/en, 18 RMB, single chapter).

## Setup commands

- Open project: launch Godot 4.3-stable editor, **Import** this folder, then **Project Manager → Edit**.
  Verified binary paths: `C:\Users\Administrator\godot_console.exe` and
  `D:\迅雷下载\Godot_v4.3-stable_win64.exe`. `project.godot` declares `features=4.6` but
  the project is verified on 4.3-stable; the API surface used is compatible.
- Run game (windowed):  `godot_console.exe --path .`
- Run game (headless):   `godot_console.exe --headless --path .`
- Run all headless tests (19 suites, ~593 assertions — the canonical regression gate):
  ```powershell
  $exe = "C:\Users\Administrator\godot_console.exe"
  foreach ($t in @(
      "save_test","sfx_test","flow_integration_test","night_shift_basic_test",
      "night_shift_data_validate","hotspot_dot_test","day_effects_test",
      "late_hotspot_enemy_test","night_report_stats_test","radio_contact_test",
      "night_shift_full_flow_test","signal_catalog_test","menu_ui_test",
      "save_slots_test","tutorial_test","i18n_test","locale_e2e_test",
      "night_shift_fx_test","ampersand_lint_test"
  )) { & $exe --headless --path . --script "res://tools/$t.gd" }
  ```
  `ampersand_lint_test` is a defensive CI gate — fails if any `.gd` file under
  `res://scripts/` or `res://tools/` contains the `&&` operator (project style
  is the `and` keyword). Expected baseline: zero hits.
- Run a single test:    `godot_console.exe --headless --path . --script res://tools/<name>.gd`
- Capture visual screenshots:  `godot_console.exe --path . --script res://tools/capture_<name>.gd`
- Release build:        `tools/build_release.ps1` (Windows) / `tools/build_release.sh` (macOS/Linux)
  — runs tests → stamps version → exports via `export_presets.cfg` → writes checksums.

## Project layout

- `scenes/` — Godot `.tscn` files (12). `NightShiftGame.tscn` is the main scene
  (`run/main_scene` in `project.godot`); the rest are screens (radio, dispatch,
  night report, settings, end-of-day, final score, base board, city map, etc.).
- `scripts/` — GDScript (28 files). `NightShiftGame.gd` is the data-driven thin
  controller / state machine; subsystem modules are split by responsibility
  (`NightShiftData`, `NightShiftLevels`, `NightShiftArt`, `NightShiftSave`,
  `NightShiftDayEffects`, `NightShiftSfx`, `NightShiftActors`, `HotspotDot`,
  `I18n`, `Settings`, `Steamworks`).
- `data/night_shift/` — formal game data (JSON): `resources.json`, `day_cards.json`,
  `chapter_01_nights.json`, `signals.json`.
- `data/i18n/` — locale strings: `zh.json`, `en.json` (paired).
- `data/defense_*.json` — legacy / upcoming second-mode data (defense game prototype).
- `tools/` — test suites, capture tools, build scripts, asset-pipeline scripts.
  Tests live alongside their capture tools; archived v0.5 scripts are prefixed
  `_archived_` and carry SKIP banners.
- `docs/` — design + release docs. Read these before changing game content:
  - `docs/LAST_RADIO_V2_DESIGN.md` — top-level design overview
  - `docs/release_roadmap.md` — milestone tracker (M1–M10) and current focus
  - `docs/radio_design.md` — radio dial minigame mechanics
  - `docs/design/game_design_spec_zh.md` — full design spec (zh)
  - `docs/design/chapter_01_night_plan_zh.md` — chapter 1 night-by-night plan (zh)
- `assets/` — `assets/final/` (formal AI-generated art, tracked) +
  `assets/fallback/` (procedural placeholder). Subdirs: `audio/`, `final/`,
  `player_walk/`, `nora_walk/`, `elias_walk/`. PNG import files (`.png.import`)
  **must be committed** alongside the PNGs so the editor opens without a re-import pass.
- `addons/godot_ai/` — optional editor plugin exposing a Godot MCP server on
  `http://127.0.0.1:8000/mcp`. Requires `uv` on first enable. Not used in CI.
- `build/` — release artifacts (gitignored, regenerated by `build_release.*`).
- `screenshots/` — capture-script output (gitignored, debug-only).

## Code style

- GDScript, 4-space indent, snake_case for vars/funcs, PascalCase for classes/nodes.
- State machine lives in `NightShiftGame.gd`; new top-level states go there and
  route to the appropriate subsystem module — don't grow new sibling controllers.
- Data → JSON in `data/night_shift/` and `data/i18n/`. Hardcoded gameplay
  constants in scripts are a regression — read the JSON instead.
- Every script carries a `.gd.uid` (Godot 4.x generates this); do not delete them.
- User-facing strings: always `tr("key")` via the `I18n` module — never inline
  hardcoded zh/en text in `scripts/` or scenes. New keys go in **both**
  `data/i18n/zh.json` and `data/i18n/en.json`.
- File operations: use Read/Write/Edit tools (UTF-8 safe) instead of
  `Get-Content | Set-Content` — the latter silently corrupts CJK on
  PowerShell 5.1. There is a real prior incident (see `CHANGELOG.md` 0.5.0 →
  Fixed → "Encoding corruption in 150+ Chinese strings").

## Testing instructions

- The 18 test suites in `tools/` are the regression gate. Every new gameplay
  behavior ships with a matching `*_test.gd` next to the implementation.
- New visual work ships with a matching `capture_*.gd` that renders the
  relevant screens to `screenshots/` for visual review.
- Bug fixes add a regression test that fails on `main` and passes on the branch.
- Tests run headless; capture scripts need the rendering backend (drop `--headless`).
- Archived v0.5 scripts (`tools/_archived_*.gd`) reference removed `_debug_*` APIs
  and must not be revived; add a new test instead.

## PR & commit conventions

- Default branch is `master` (not `main`).
- Branch from `master`; never push to it directly — open a PR.
- Commit message style: conventional commits (`feat:`, `fix:`, `chore:`,
  `refactor:`, `docs:`, `test:`). `tools:` prefix for build/asset-pipeline work.
- Tag format after each M completion: `v0.x-m{N}-done` (per
  `docs/release_roadmap.md`).
- Open the PR via `gh pr create` once tests pass and the diff is review-ready.
- Scope: keep PRs small and topical. Save-schema changes need a migration note
  in the PR body; i18n changes need both `zh.json` and `en.json` updated.

## Security

- Steamworks integration (`scripts/Steamworks.gd`) is a stub — do not commit
  real Steam app keys, achievement IDs, or cloud-save credentials.
- Cloud-save sync (M4) is stubbed; user save files live under `user://` and
  never get committed.
- No telemetry is collected client-side. The `godot_ai` MCP plugin runs
  locally on `127.0.0.1:8000` only; do not expose it to the network.
- `LICENSE` is MIT. Third-party attributions in `THIRD_PARTY.md` must stay
  current when new assets or libs are added.
